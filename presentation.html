01

02
Pulled top 30 contributors per repo via GitHub Contributors API
Cutoff at 30: beyond this, single-commit drive-by contributors dominate
03
Built bipartite graph (repos ↔ contributors) → projected onto repo-repo network
Edge weight = number of shared contributors between two repos

Network Structure · Degree Distribution

A Bimodal Distribution Reveals Core-Periphery Structure

What we see
Two distinct peaks: most repos cluster at low degree (0–25), but a second group clusters near degree 120–140. This is not a typical power-law.
Interpretation
The ML ecosystem has a core-periphery structure: a dense hub of foundational repos whose contributors circulate freely between each other, surrounded by a larger periphery of specialised repos.
Network stats
Mean degree: 43.5 · Max: 165 · Density: 0.072

Hypothesis Test · Two-Sample t-Test

Network Position and Popularity are Independent

H₀ (null)
Core and periphery repos have equal mean log-stars
H₁ (alternative)
Core repos are significantly more popular than periphery repos
0.2396
p-value · t = 1.18 · df = 410
Result: fail to reject H₀

Being structurally central does not predict popularity. Popularity (stars) and network position are independent dimensions — a repo can be famous without being a contributor hub, and vice versa.

10
Conclusion

Three Answers, One Coherent Picture

Non-random structure confirmed. Clustering 5.3× above the ER null model — contributors form overlapping circles, not random connections. The ecosystem has genuine internal organisation.
Communities exist but are porous. 11 communities detected (Q=0.209). Community 0 — TensorFlow, PyTorch, HuggingFace, OpenCV — forms the foundational core, but low modularity shows contributors cross boundaries freely. ML is one ecosystem with sub-tendencies, not isolated silos.
Popularity ≠ centrality. t-test p=0.24: network position and GitHub stars are statistically independent. Being a contributor hub and being famous are different things.
